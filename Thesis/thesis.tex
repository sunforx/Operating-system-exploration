\Abstract{
操作系统最初的诞生是为了搭配大型机进行各种简单繁重的数字运算，但随着时代的演进，
计算机已经不仅仅作为处理数字运算的机器，其附加价值也越来越被人们看重，
人们在生活中赋予了计算机各种各样的角色：电话，电视机，游戏机等，
使计算机通过软件方式来提高自己的做事效率以及丰富个人的生活。
多道程序系统，分时操作系统，一个个新技术的诞生标志着操作系统越发的完善，
计算机操作系统的体积也不断增大。
但是计算机的发展似乎到了一个瓶颈，工程师不断将各种各样的功能代码加入到系统内核中，
导致系统内核代码量成倍的增长，出现BUG的几率也成倍的增长，
所以，为了探究如何简化操作系统，那么就应该拨开操作系统华丽的外衣，
从历史的迷雾中寻找操作系统的本质，并从底层开始，不断探索并实现各项功能，
最简洁的代码完成一个满足现代各项需求的操作系统。
  }

\enTitle{Operating system exploration}
\enAuthor{Zach Yin}
\enAbstract{
  % The initial version of the operating system was to collaborate with simple and heavy-duty digital computing machines. 
  % However, with the evolution of the times, computers have not only been used as machines for processing various operations, 
  % but also added more value with people. With the development of computers, The multi-programming system, 
  % the time-sharing operating system, and the emergence of new technologies mark the increasingly perfect operating system. 
  % However, to explore the operating system, it should not satisfy the existing operating system. 
  % It should open up the gorgeous operating system. Starting from the ground floor, 
  % after a step-by-step exploration, we constantly explored and implemented various functions, 
  % completing an operating system that satisfies modern requirements from the most concise code.

  The initial operation of the operating system was to collaborate with the mainframe to perform various simple and heavy-duty digital operations, 
  but as the times evolved, Computers have not only been used as machines for digital computing, but their added value has also been valued by people.
  People have given the computer a variety of roles in life: telephones, television sets, game consoles, etc.
  Make computers use software to improve their efficiency and enrich their lives.
  The multi-programming system, the time-sharing operating system, and the birth of new technologies mark the increasingly perfect operating system.
  Computer operating systems are also growing in size.
  However, the development of computers seems to be a bottleneck. Engineers continue to add a variety of function codes to the system kernel.
  As a result, the amount of code in the system kernel has increased exponentially, and the probability of a BUG has also multiplied.
  So, in order to explore how to simplify the operating system, you should open up the gorgeous coat of the operating system.
  Look for the essence of the operating system from the fog of history and start from the bottom, constantly exploring and implementing various functions.
  The simplest code completes an operating system that meets all modern requirements.
  }
